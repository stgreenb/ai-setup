# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/rely-ai-org/caliber) — do not edit manually.

- **[pattern]** When closing duplicate/inferior PRs, always leave a comment thanking the contributor, naming which PR was chosen instead, and explaining the specific technical reason (e.g., "#42 uses console.warn which is testable; yours uses process.stderr.write which can't be spied on in vitest").
- **[gotcha]** Issues linked to PRs may auto-close when the PR merges (if PR body has "Fixes #N"). Check with `gh issue view` before manually closing — you'll get "is already closed" if it was auto-closed.
- **[pattern]** For demo GIF recordings of `caliber score`, use `Set FontSize 12`, `Set Width 900`, `Set Height 1100` — the score output is ~64 lines tall, so a tall narrow terminal fits everything without clipping.
- **[gotcha]** VHS recordings that invoke LLM-backed commands (`caliber init`, `caliber refresh`) can hang indefinitely in the headless browser terminal. Pre-stage results using git tags (e.g., `before-init`, `after-init`) and only run deterministic commands (`caliber score`) live in the tape. For LLM commands, use `expect` scripts outside VHS or pre-commit the results.
- **[pattern]** For demo recordings, use `expect` (macOS built-in at `/usr/bin/expect`) to automate `caliber init` interactive prompts — match on prompt text like `"coding agents"`, `"session learning"`, `"What would you like to do"` and send `"a"` (select all) or `"\r"` (accept default). Set timeout to 180s for LLM-heavy flows.
- **[gotcha]** Always run VHS from the caliber source directory, not the demo repo — VHS `Output` paths are relative to CWD, and running from the wrong directory puts output files in unexpected locations.
- **[gotcha]** On macOS, common shell aliases like `rm -i` and `cp -i` cause `rm` and `cp` to prompt interactively, which hangs non-interactive shells. Use backslash-escaped commands (`\rm -f`, `\cp`) to bypass aliases in scripts and automated recordings.
- **[fix]** Before re-recording VHS tapes that switch git branches in the demo repo, clean the repo state first (`git checkout -- . && git clean -fd -q`) — otherwise dirty working tree from a previous recording causes checkout failures that silently break the tape.
- **[gotcha]** `npx create-video@latest` (Remotion scaffold) prompts interactively for options like TailwindCSS, which hangs non-interactive shells. Instead of using the scaffold, manually create `assets/video/` with `package.json`, `tsconfig.json`, and component files.
- **[pattern]** The Remotion video project at `assets/video/` has its own `package.json` and `pnpm-lock.yaml` — run `pnpm install` inside `assets/video/` before building. Build commands: `remotion render CaliberDemo assets/demo.mp4 --codec h264` for MP4, `--codec gif --every-nth-frame 2` for GIF.
- **[pattern]** When closing duplicate/inferior PRs, always leave a comment thanking the contributor, naming which PR was chosen instead, and explaining the specific technical reason.
- **[fix]** `gh pr create` without `--repo` fails when git remote points to `rely-ai-org/caliber` but the actual GitHub repo is `caliber-ai-org/ai-setup` — always use `--repo caliber-ai-org/ai-setup` or update the remote URL first with `git remote set-url origin`.
- **[gotcha]** GitHub `<video>` tags in README.md do NOT render in the main repo view — GitHub strips `<video>` from rendered markdown. Use `<img src="...gif">` for animated content in README, or link to the MP4 separately. The `<video>` tag only works in GitHub Issues, PRs, and Discussions.
- **[gotcha]** `WebFetch` from `raw.githubusercontent.com` can silently return empty responses for some SVG files (e.g., `openai.svg`). Use `cdn.jsdelivr.net/npm/simple-icons@latest/icons/<name>.svg` via `curl` as a reliable fallback for fetching simple-icons SVGs.
- **[gotcha]** Next.js RSC pages return serialized component data via `WebFetch`, not rendered HTML — you cannot extract Tailwind classes or DOM structure. To get design tokens, fetch the CSS file directly from the `/_next/static/chunks/*.css` URL found in the page source.
- **[gotcha]** Running `pnpm test` or `npx vitest` produces empty/silent failures when `pnpm install` hasn't been run after a fresh checkout or branch switch. Always run `pnpm install` before running tests if dependencies might be stale.
- **[pattern]** Scoring checks use `git check-ignore` (not `git ls-files`) to detect gitignored directories — it's a targeted check on specific paths rather than listing all tracked files, which scales better on large repos.
- **[gotcha]** `caliber learn finalize` may fail silently with a stale lock file (`.caliber/learning/finalize.lock`) if a previous session crashed. The lock persists even after the process dies. Remove it manually with `rm .caliber/learning/finalize.lock` before retrying `caliber insights` or `caliber learn finalize`.
- **[fix]** If `caliber learn finalize` crashes with "content.split is not a function", the issue is often in JSON response normalization — the LLM may return `claudeMdLearnedSection` as an array instead of a string. Add a `normalizeAnalysisResult()` function to convert arrays to strings and coerce non-string values before parsing.
- **[pattern]** When `caliber init` installs learning hooks, use bare `caliber` commands (e.g., `caliber learn finalize --auto`) in `.claude/settings.json` hooks instead of absolute paths. Absolute paths (e.g., `/Users/alonpe/.nvm/versions/node/v20.20.1/bin/caliber`) break across machines and Node version upgrades. The `resolveCaliber()` function should prefer bare command names when the binary exists on PATH.
- **[gotcha]** Dependencies must be installed before running build or test commands. Fresh checkouts, branch switches, or after dependency changes require `npm install` (or `pnpm install`) before `npm run build`, `npx vitest`, or `npx tsc` will work. Missing node_modules causes "command not found" errors even if tools are in package.json.
- **[pattern]** Terminology shift in UI: changed from "setup" to "config" across user-facing messages and test expectations. This is a branding/UX improvement — when updating docs or tests, use "config" not "setup" for consistency with current messaging.
- **[fix]** When test assertions fail due to changed UI strings, update both the implementation (e.g., `displayColdStart()` messaging) AND the corresponding test expectations. Tests like `src/commands/__tests__/insights.test.ts` must mirror the new output strings.
- **[pattern]** Score history tracking: record scores at key decision points (`init`, `regenerate`, `refresh`, `score`, `manual` triggers) into `.caliber/score-history.jsonl` for trend analysis and ROI calculation. Use `recordScore(result, 'trigger')` to append JSONL entries.
- **[pattern]** Quality gates on refresh: snapshot pre-refresh scores and file contents, compute post-refresh scores, and revert all changes if the score regressed. This prevents LLM-generated updates from degrading config quality. Store snapshots in a Map and restore on failure.
- **[pattern]** Rate-limit refresh operations with a 30-second cooldown (`REFRESH_COOLDOWN_MS = 30_000`). Check `state?.lastRefreshTimestamp` before running to prevent excessive LLM calls when running refresh in tight loops (e.g., from CI or hooks).
- **[pattern]** Retry LLM calls once on transient failure in `refresh` command — catch first error, retry the same payload, then fail with the original error if retry fails. This improves resilience in hook-triggered refreshes which run silently.
- **[pattern]** Insights command displays config score trends from history: read `readScoreHistory()`, compute trend direction (`up`/`down`/`stable`), and show delta with arrow symbol (\u2191/\u2193/\u2192). Helps users visualize whether their config quality is improving.
- **[pattern]** Cold-start insights messaging: when no learning hooks are installed, explain briefly what session learning does (captures patterns from coding sessions) and how to enable it. Prompt clarity improves adoption.
- **[pattern]** Early-stage insights (1-19 sessions): show progress tracker ("X/20 sessions tracked") to set expectations for when full insights unlock. Reduces confusion about when ROI data becomes meaningful.
